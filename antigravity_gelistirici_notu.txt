  BÖLÜM 1 — SİSTEM GENEL BAKIŞ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Bu not, "Antigravity" kod adıyla geliştirilen tarayıcı tabanlı manga çeviri
editörünün mevcut durumunu, teknik mimarisini ve gelecekteki geliştirme
önerilerini özetlemektedir. Sistemi devralacak veya üzerine inşa edecek bir
yapay zeka ya da geliştirici için referans niteliği taşımaktadır.


[ TEKNİK MİMARİ ]

  - Tamamen istemci taraflı (client-side) çalışan tek sayfalık web uygulaması
  - Temel dosya: app2.js
  - Backend sunucusu gerektirmez — tarayıcıda koşar
  - Çeviri motoru: Google Gemini 2.5 Flash (varsayılan)
  - Döviz kuru kaynağı: Frankfurter API (canlı USD/TRY)


[ TAMAMLANAN ÖZELLİKLER ]

  OCR & Çeviri Pipeline
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
  · Manga sayfası görseli API'ye gönderilir; model hem metni okur hem Türkçe'ye
    çevirir
  · Her istek için token kullanımı takip edilir
  · Maliyet canlı döviz kuruyla TL cinsinden ekrana yansıtılır

  Canvas Tabanlı Editör
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
  · Metin blokları sürüklenebilir ve yeniden boyutlandırılabilir
  · Yazı tipi, boyut, renk, hizalama ayarları mevcut
  · Seçili bloğu silme ve sayfayı export etme destekleniyor

  Çok Sayfalı Destek
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
  · Birden fazla manga sayfası aynı oturumda yönetilebilir
  · Sayfalar arası geçişte canvas durumu korunur

  Tamamlanan Hata Düzeltmeleri (6 Kritik Patch)
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
  · Türkçe büyük harf dönüşümü (İ, Ş, Ğ vb.) düzeltildi
  · Canlı döviz kuru entegrasyonu eklendi (statik değer kaldırıldı)
  · Canvas render sırası ve z-index çakışmaları giderildi
  · UI/UX tutarsızlıkları (buton durumları, yükleme göstergeleri) düzeltildi

  BÖLÜM 2 — DİKKAT EDİLMESİ GEREKENLER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  API & Güvenlik
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄
  · Gemini API anahtarı şu an istemci tarafındadır — tarayıcı geliştirici
    araçlarından görülebilir. Kullanıcı sayısı artarsa bir proxy sunucusunun
    arkasına alınmalıdır.
  · Frankfurter API ücretsiz ve anonimdir; üretim ortamında servis kesintisine
    karşı fallback bir sabit kur değeri tanımlanmalıdır.

  Canvas & Performans
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
  · 20+ sayfalık projelerde canvas nesneleri bellek baskısı oluşturabilir.
    Pasif sayfalar için lazy unload mekanizması düşünülmelidir.
  · Metin blok koordinatları yalnızca oturum içinde tutulmaktadır;
    sayfa yenilenirse tüm düzenlemeler kaybolur.

  Türkçe Dil Özellikleri
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
  · Modeller manga argosunu ve onomatopeia'ları (BAAAM, ZAP vb.) zaman zaman
    düzleştirebilir. Kritik sayfalar için manuel gözden geçirme ihmal edilmemeli.
  · Font seçiminde Türkçe karakter desteği (İ, Ö, Ü, Ş, Ğ, Ç) zorunludur;
    Latin-only fontlar karakter bozukluğuna yol açar.

  BÖLÜM 3 — ÖNERİLEN EKLEMELERi
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  >> KISA VADELİ (Düşük Efor / Yüksek Değer)

  · Otomatik kaydetme      — localStorage ile canvas durumunu periyodik kaydet
  · Geri al / İleri al    — Komut geçmişi stack'i ile editör hataları geri alınabilir
  · Toplu export           — Tüm sayfaları ZIP içinde PNG/JPG olarak indirme
  · Bubble maskeleme       — Orijinal metni beyaza boyayan overlay önizleme modu

  >> ORTA VADELİ (Orta Efor / Yapısal Değer)

  · Proje dosyası (.json)  — Tüm düzenleri dışa/içe aktarılabilir formatta sakla
  · OCR bölge seçimi       — Sayfanın sadece belirli bölgesini API'ye gönder;
                             token tasarrufu sağlar
  · Çeviri Belleği (TM)   — Tekrar eden terimleri (karakter isimleri, özel
                             terimler) sözlükte tutarak tutarlı çeviri sağla
  · Font embedding         — Türkçe manga estetiğine uygun özel font paketi

  >> UZUN VADELİ (Yüksek Efor / Platform Değeri)

  · Hafif backend          — API anahtarını güvene al, kullanıcı bazlı proje yönetimi
  · Glossary UI            — "Bu karakter her zaman X olarak çevrilsin" kuralları
  · Manga serisi yönetimi  — Bölüm organizasyonu, ilerleme ve çeviri durum panosu
  · Gemini batch modu      — Birden fazla sayfayı tek API çağrısında işle

  BÖLÜM 4 — OPENROUTER MODEL ÖNERİLERİ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Antigravity sistemi için bir modelin uygun sayılabilmesi için üç temel
  kriteri karşılaması gerekir:

    1. Görüntü girişi desteklemeli  (Vision / Multimodal)
    2. Japonca/Korece → Türkçe çeviri kalitesi yüksek olmalı
    3. Token başına maliyet makul seviyede olmalı

  Aşağıdaki modeller OpenRouter üzerinden erişilebilir ve bu kriterlere göre
  değerlendirilmiştir.

  BÖLÜM 5 — SON NOTLAR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  · Sistem bağımsız ve taşınabilir olacak şekilde tasarlanmıştır; bu mimariyi
    korumak uzun vadede bakım maliyetini düşürür.
  · Token maliyet takibi son derece değerlidir. İleride haftalık/aylık rapor
    olarak kullanıcıya sunulması faydalı olabilir.
  · Sistem şu anki haliyle tek kullanıcılı profesyonel bir araç olarak
    olgunlaşmış durumdadır. Bir sonraki eşik, veriyi oturum dışına taşımak
    ve çeviri tutarlılığını sistematik hale getirmektir.
  · OpenRouter'ın unified API yapısı büyük avantajdır: tüm modeller aynı
    endpoint üzerinden çağrılabilir, model değişikliği yalnızca bir string
    değişikliğine indirgenir. Bu nedenle model seçici eklemenin teknik maliyeti
    son derece düşüktür.
